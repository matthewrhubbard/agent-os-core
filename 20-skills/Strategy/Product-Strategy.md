# Product Strategy

Use this skill when diagnosing or designing product strategy across the main levers of an existing or emerging product.

## Purpose

This skill helps determine where product strategy effort should go by separating four major domains:

1. Features
2. Growth
3. PMF expansion
4. Scalability

It is especially useful for product leaders who need to decide whether the right next move is improving the core product, growing distribution, expanding product-market fit, or investing in scale.

## Context Check

Before using this skill:

- Identify the current product and stage
- Confirm whether the product has meaningful product-market fit already
- Identify the current growth, retention, and product constraints
- Identify the relevant source-of-truth files, metrics, and decisions
- If key context is missing, ask one targeted question with likely options, brief tradeoffs, and a recommended default, or state assumptions explicitly when reasonable

## Product Strategy Frame

Evaluate the situation through four lenses:

1. Feature strategy
2. Growth
3. PMF expansion
4. Scalability

The main question is not "What should we build next?" The main question is "Which of these four domains is the real strategic bottleneck right now?"

## 1. Feature Strategy

Use this lens when the main opportunity is to improve the existing value proposition.

Ask:

- Which target population matters most?
- What user value is being created?
- What business value is being created?
- Which existing or proposed features are driving adoption, retention, and satisfaction?
- Are we adding features before fully understanding the value of what already exists?

Use this when the core product still has obvious room to improve before broader expansion.

## 2. Growth

Use this lens when the product may already be valuable, but distribution or usage growth is the main constraint.

Ask:

- Are acquisition loops strong enough?
- Are retention loops healthy?
- Is monetization underdeveloped relative to user value?
- Are we trying to grow before we have real repeat value?

Use this when the issue is not mainly feature gaps, but weak distribution, retention, or monetization mechanics.

## 3. PMF Expansion

Use this lens when the current product-market fit may be saturating, degrading, or limited.

Ask:

- Has the current market saturated?
- Has the current product been optimized enough for the initial segment?
- Is PMF degrading due to competition or over-optimization?
- Should we adapt the current product to adjacent markets, geographies, or use cases?
- Is there a complementary expansion worth pursuing?
- Is this early enough that expansion would be premature?

Use this when the question is how to extend a working product into adjacent demand, not merely improve the current experience.

## 4. Scalability

Use this lens when the product or organization is straining under success or complexity.

Ask:

- Are technical systems becoming the bottleneck?
- Is process scaling or organizational coordination failing?
- Is user behavior creating new forms of load or risk?
- Is technical debt now constraining strategic options?
- What should be modernized now versus deferred?

Use this when scaling problems are starting to shape product decisions.

## Choosing The Right Lens

Do not assume the answer is new features.

A product strategy diagnosis should first ask:

- Is the main problem product value?
- Is it growth?
- Is it PMF expansion?
- Is it scale?

The right strategic move depends on the real bottleneck.

## Output Structure

Use this shape:

1. Product stage
2. Main strategic bottleneck
3. Best-fit lens: features / growth / PMF expansion / scalability
4. Evidence supporting that diagnosis
5. Tradeoffs and alternatives
6. Recommended next moves
7. Key risks or sequencing constraints

## Quality Bar

A strong product strategy analysis:

- correctly identifies the current product stage
- does not confuse roadmap ideas with strategy
- chooses the right strategic lens
- shows why other lenses are not primary right now
- ties recommendations to evidence and sequencing

## Common Failure Modes

- assuming every problem is a feature problem
- trying to grow before core value is strong
- expanding PMF too early
- investing in scale before it matters
- confusing volume of ideas with strategic clarity
