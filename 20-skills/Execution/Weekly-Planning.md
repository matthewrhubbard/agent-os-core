# Weekly Planning

Use this skill to turn priorities into a realistic weekly plan that reflects time, energy, constraints, and sequence.

## Purpose

This skill helps bridge strategy and execution by deciding what should actually happen this week.

- `../../00-core/01-Operating-Principles.md`

## Context Check

Before using this skill:

- Identify the relevant workstreams or life areas for the week
- Confirm current priorities and deadlines
- Check calendar constraints, travel, or other capacity limits
- Review open loops from the previous week
- If key context is missing, ask one targeted question with likely options, brief tradeoffs, and a recommended default, or state assumptions explicitly when reasonable

## When To Use

- At the start of a week
- After priorities change midweek
- When the week feels overloaded or unclear

## Core Principles

- Plan for reality, not idealized capacity
- Sequence matters
- A smaller plan that gets done beats an aspirational plan that collapses
- Protect time for the highest-leverage work

## Workflow

## 1. Review The Starting Point

Look at:

- Top priorities
- Existing commitments
- Open loops
- Deadlines and dependencies

## 2. Define The Week's Wins

Choose the few outcomes that would make the week successful.

## 3. Allocate Work Realistically

Map:

- Must-do items
- Important but flexible items
- Low-priority or deferrable items
- Buffer for surprises

## 4. Sequence The Work

Decide:

- What needs focused time
- What can be batched
- What can be delegated, deferred, or dropped

## Output Structure

1. Weekly context and constraints
2. Top outcomes for the week
3. Must-do work
4. Secondary work
5. Risks or overload points
6. Recommended sequencing

## Quality Bar

A good weekly plan:

- Reflects actual capacity
- Protects the most important work
- Anticipates friction
- Is specific enough to use day to day

## Common Failure Modes

- Planning more than capacity allows
- Confusing activity with outcomes
- Ignoring the calendar
- Leaving no margin for interruptions
