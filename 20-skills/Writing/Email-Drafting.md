# Email Drafting

Use this skill to write emails that are clear, appropriately toned, and designed to get the intended outcome.

## Purpose

This skill helps draft emails that match the relationship, context, and ask instead of sounding generic or overlong.

Related files:
- `../../00-core/02-Communication-Style.md`
- `../../15-people/People-Index.md`

## Context Check

Before using this skill:

- Identify the current project, domain, or life area
- Confirm the audience and relationship
- Clarify the desired outcome of the email
- Check for any relevant people notes or prior thread context
- If key context is missing, ask one targeted question with likely options, brief tradeoffs, and a recommended default, or state assumptions explicitly when reasonable

## When To Use

- Drafting a new email
- Revising an email for tone or clarity
- Writing follow-ups, asks, or stakeholder updates

## Core Principles

- Be clear about the purpose early
- Match tone to the relationship and stakes
- Make the ask specific
- Respect the reader's time
- Do not bury the action needed

## Workflow

## 1. Define The Objective

Clarify:

- Why this email is being sent
- What response or action is desired
- What tone is appropriate

## 2. Draft For The Reader

Write with:

- A clear opening
- Useful context only
- A direct ask
- A clean close

## Output Structure

1. Subject line options if useful
2. Draft email
3. Brief note on tone or recommendation if helpful

## Quality Bar

A strong email draft:

- Is easy to understand on first read
- Sounds like it was written for the actual recipient
- Makes the desired next step obvious
- Is concise without being abrupt

## Common Failure Modes

- Writing for yourself instead of the reader
- Overexplaining
- Hiding the ask
- Using the wrong tone for the relationship
