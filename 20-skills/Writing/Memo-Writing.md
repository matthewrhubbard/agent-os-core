# Memo Writing

Use this skill to write a memo that informs, frames a problem, or recommends a course of action with strong structure and clarity.

## Purpose

This skill helps produce decision-useful memos rather than long documents that are technically complete but hard to act on.

## Context Check

Before using this skill:

- Identify the current project, domain, or life area
- Confirm the audience and the memo's job
- Identify the source-of-truth files or evidence base
- Check whether the memo is informative, analytical, or persuasive
- If key context is missing, ask one targeted question with likely options, brief tradeoffs, and a recommended default, or state assumptions explicitly when reasonable

## When To Use

- Writing a decision memo
- Summarizing research or recommendations
- Framing a problem for a stakeholder

## Core Principles

- Lead with the point
- Structure around decisions, insights, or recommendations
- Use only the context needed to support action
- Make tradeoffs and uncertainty visible

## Workflow

## 1. Define The Memo's Job

Clarify whether the memo should:

- Inform
- Recommend
- Diagnose
- Compare options

## 2. Build The Structure

Most memos should include:

- Purpose
- Key conclusion or recommendation
- Supporting context
- Tradeoffs or risks
- Next steps

## Output Structure

1. Title
2. Executive summary
3. Main sections
4. Recommendation or takeaway
5. Next steps if relevant

## Quality Bar

A good memo:

- Has a clear point
- Is easy to scan
- Supports conclusions with evidence or reasoning
- Helps the reader decide or act

## Common Failure Modes

- Starting with background instead of the point
- Hiding the recommendation
- Including context that does not change action
- Sounding polished but saying little
